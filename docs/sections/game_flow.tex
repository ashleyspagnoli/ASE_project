\section{API Calls of the Game Flow}
All the following requests require the \texttt{Authorization} header containing the JWT token of the authenticated user:
\begin{verbatim}
Authorization: Bearer <your_jwt_token>
\end{verbatim}

\subsection{Deck Building Phase}
We've separated the deck building from the game (More information in \hyperref[sec:cloud-decks]{Cloud Storage of Decks})
\begin{itemize}
    \item \textbf{GET /collection/cards}: Retrieve all available cards.
    \item \textbf{POST /collection/decks}: Create a new deck.
    \begin{verbatim}
    Body:
    {
      "deckSlot": 1,
      "deckName": "My Deck",
      "cards": ["hA", "h5", "d5", "dK", "c7", "c8", "sA", "s5"]
    }
    \end{verbatim}
    \item \textbf{GET /collection/decks}: Retrieve the user's decks.
\end{itemize}

\subsection{Game Phase}
\begin{itemize}
    \item \textbf{POST /game/match/join}: Join the matchmaking queue.
    \item \textbf{GET /game/match/status}: Check the status of the matchmaking process.
    \item \textbf{POST /game/deck/\{game\_id\}}: Submit the chosen deck for the game.
    \begin{verbatim}
    Body:
    {
      "deck_slot": 1
    }
    \end{verbatim}
    \item \textbf{GET /game/hand/\{game\_id\}}: Retrieve the player's current hand.
    \item \textbf{POST /game/play/\{game\_id\}}: Execute a play (e.g., play a card).
    \begin{verbatim}
    Body:
    {
      "card": {
        "value": "K",
        "suit": "clubs"
      }
    }
    \end{verbatim}
    \item \textbf{GET /game/state/\{game\_id\}}: Retrieve the current state of the game.
\end{itemize}
